Learning PathA structured study order — SOLID first, then patterns by industry relevance, with case studies interleaved at each stage

SOLID Principles 5 topics
The foundation everything else builds on. Patterns are solutions to problems SOLID describes — you can't appreciate Factory Method without understanding DIP.

Tier 1 — Gateway Patterns 5 patterns
After this phase, you can build a clean service with configurable behavior, event handling, and proper object creation. Strategy replaces if/else, Singleton teaches why DI exists.
1. Strategy 2. Singleton 3. Factory Method 4. Observer 5. Template Method
3
Tier 2 — Structural Essentials 5 patterns
ASP.NET middleware IS Decorator. Every service class is a Facade. EF lazy loading uses Proxy. These are the patterns you'll touch daily.
1. Decorator 2. Adapter 3. Facade 4. Proxy 5. Composite
4
Tier 3 — Core Creational & Behavioral 5 patterns
State machines power half the case studies. Command enables undo/redo and CQRS. Abstract Factory and Builder handle complex object creation.
1. State 2. Command 3. Abstract Factory 4. Builder 5. Prototype
Starter Case Studies (9) — apply your first 15 patterns

Tier 4 — Advanced Behavioral 4 patterns
Middleware pipelines (Chain of Resp), MediatR hubs (Mediator), LINQ under the hood (Iterator), and serialization snapshots (Memento).
1. Chain of Resp 2. Mediator 3. Iterator 4. Memento
Intermediate Case Studies (21) — multi-pattern systems

Tier 5 — Specialized 4 patterns
Platform abstractions (Bridge), memory optimization (Flyweight), Roslyn syntax walkers (Visitor), regex/DSLs (Interpreter). Learn these as needed.
1. Bridge 2. Flyweight 3. Visitor 4. Interpreter
